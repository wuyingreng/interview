🎯 缓存策略说明

🔒 强缓存 (Cache First)

优先使用缓存，适用于静态资源和不经常变化的内容。

适用场景:

  • JavaScript 文件
  • CSS 样式文件
  • 图片资源
  • 字体文件
  • 版本化资源

🔄 协商缓存 (Network First)

优先请求网络，失败时使用缓存，适用于动态内容。

适用场景:

  • API 接口数据
  • 用户生成内容
  • 配置信息
  • 实时数据
  • HTML 页面

🌐 仅网络 (Network Only)

只使用网络请求，不缓存任何内容，适用于敏感操作。

适用场景:

  • 用户登录
  • 支付操作
  • 文件上传
  • 管理操作
  • 敏感数据

💾 仅缓存 (Cache Only)

只使用缓存，不请求网络，适用于离线模式。

适用场景:

  • 离线模式
  • 预加载资源
  • 关键资源
  • 备用内容
  • 降级方案

📚 使用指南

何时使用强缓存？

  • 静态资源：JS、CSS、图片、字体等不经常变化的文件
  • 版本化资源：带有 hash 值的文件名，内容变化时文件名也会变化
  • 离线优先：需要快速加载的关键资源
  • 性能优化：减少网络请求，提升加载速度

何时使用协商缓存？

  • API 数据：需要实时性的接口数据
  • 动态内容：用户生成的内容、评论、动态等
  • 配置信息：可能变化的系统配置
  • 平衡性能：在性能和实时性之间取得平衡

最佳实践建议

  • 根据资源类型选择合适的缓存策略
  • 设置合理的缓存过期时间
  • 定期清理过期缓存
  • 监控缓存命中率和性能指标
  • 提供离线降级方案
